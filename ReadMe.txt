1 版本第一次提交说明：
项目的主要步骤：
1）海康初始化。
2）海康登录。
3）海康播放。分为直接窗口播放和回调数据后自己处理数据播放，回调播放也可
以分为使用播放库Play和不使用播放库，不使用时需要调用ffmpeg自己解析ps流。
4）海康停止播放。
5）海康退出登录。
6）海康资源清理。

该版本支持的功能：
1）支持V30,V40的海康版本播放。
2）支持直接在窗口上面播放(包括v30,v40)。
3）也支持调用回调播放(包括v30,v40)。

项目注意的地方：
1）当使用回调播放时，播放的句柄不能使用listView这些视图控件，否则无法播放，要使用Widget这个窗口子控件。

2）回调因为调用了播放库，所以在停止的时候必须也调用，否则按下停止播放画面仍在播放数秒才停。

3）不能直接在海康的回调函数中直接调用ui->widget->windId()，否则会Cannot set parent, new parent is in a different thread或者
QPainter::begin: Paint device returned engine == 0, type: 1这样的错误。

4）用于的自定义数据从fRealDataCallBack到DecCBFun可以通过long进行传参。

5）在重写的绘图事件中绘图，不能直接指定父对象为QPainter p(ui->widget-);
需要指定为QPainter p(this);否则出现QWidget::paintEngine xxx错误。实际上该绘图事件由系统不定时自动绘图，我们只需要调用update即可。

6）在回调播放时，因为无法直接在海康的回调处理数据，所以我开了一个线程来处理，在拿到yv12解压缩的数据后，
我把图片转给提前传入的线程对象，然后该线程对象发送信号给主线程的槽函数去处理，最终实现播放。

7）回调时，想要用image播放，就必须将yv12转成rgb，否则是无法播放的。

8）最后在释放海康的资源时，按下一次退出键(按下右上角的关闭不会出现这种情况)出现调用两次on_ExitButton_clicked的情况，经过调试发现，是因为我的右上角关闭按钮也指定了和退出按钮相同的槽函数。
因为即使你按下退出键，它系统内部最终都会调用右上角的按键，或者更具体说也会发送destroyed信号，导致也调用该函数。
